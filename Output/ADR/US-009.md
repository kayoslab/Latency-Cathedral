ADR-009 is required. The aggregator introduces a new coordination pattern — it is the first component that fuses multiple independent metric sources (ProbeSampler, ResourceTimingCollector) with a mode-switching mechanism (PresetState overrides live) into a single unified output. This is architecturally distinct from both the polling-based probe sampler (ADR-007) and the observer-based resource timing collector (ADR-008). The ADR at `docs/adr/009-metrics-aggregator.md` should document: (1) cadence strategy — the aggregator runs its own setInterval rather than synchronizing with ProbeSampler's interval, reading latest state on its own tick (a deliberate decoupling choice), (2) preset-override semantics — preset activation short-circuits the entire live aggregation pipeline, which is a mode-switching pattern not present elsewhere, (3) derived metric computation — packetLoss and bandwidth are computed from raw data rather than collected directly, establishing a precedent for derived metrics, (4) graceful degradation contract — the aggregator guarantees valid numeric output regardless of which sources are present, which is a stronger contract than either individual collector provides alone, and (5) polling vs subscribing trade-off — the aggregator polls presetState.current() each tick rather than subscribing reactively, accepting up to one tick of latency in exchange for simpler lifecycle management and no subscription cleanup.
